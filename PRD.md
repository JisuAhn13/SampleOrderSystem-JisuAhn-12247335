# PRD — SampleOrderSystem 제품 요구사항 문서

## 배경 및 목적

가상의 반도체 회사 **S Semi**는 연구소, 팹리스(Fabless) 업체, 대학 연구실 등의 고객에게 반도체 시료를 납품한다.
시료는 주문이 들어오면 웨이퍼 공정 설비를 통해 제작되고, 검수를 거쳐 고객에게 출고된다.
본 시스템은 이 흐름을 콘솔 기반으로 관리한다.

---

## 도메인 용어

| 용어 | 설명 |
|------|------|
| 시료 (Sample) | 반도체 웨이퍼 공정으로 만들어진 제품 단위 |
| 수율 (Yield) | 정상 시료 수 / 총 생산 시료 수 (0.0 초과 ~ 1.0 이하; 0 제외 — 0 나누기 방지) |
| 평균 생산시간 | 시료 1개 생산 소요 시간, 단위: **초(second)** |
| 부족분 | 주문 수량 - 현재 재고 수량 |
| 실 생산량 | `ceil(부족분 / (수율 × 0.9))` |
| 총 생산 시간 | `평균 생산시간 × 실 생산량` |

---

## 역할별 흐름

| 역할 | 담당 업무 |
|------|-----------|
| **고객** | 이메일로 시료 요청 |
| **주문담당자** | 요청에 맞게 주문서 작성 및 등록 |
| **생산담당자** | 주문 수신 후 승인 또는 거절, 재고 확인, 출고 준비 및 처리 |
| **생산라인** | 재고 부족 시 시료 생산 |

---

## 주문 상태 흐름

```
RESERVED → (승인) → 재고 충분 → CONFIRMED → RELEASE
RESERVED → (승인) → 재고 부족 → PRODUCING → CONFIRMED → RELEASE
RESERVED → (거절) → REJECTED  (모니터링 제외)
```

| 상태 | 의미 |
|------|------|
| `RESERVED` | 주문 접수됨 |
| `REJECTED` | 주문 거절됨 |
| `PRODUCING` | 승인 완료, 재고 부족으로 생산 중 |
| `CONFIRMED` | 승인 완료, 출고 대기 중 |
| `RELEASE` | 출고 완료 |

---

## 메인 메뉴

| 번호 | 메뉴 | 설명 |
|------|------|------|
| 1 | 시료 관리 | 시료 등록, 목록 조회, 이름 검색 |
| 2 | 주문 관리 | 주문 접수, 승인, 거절 |
| 3 | 모니터링 | 상태별 주문 현황, 재고 현황 확인 |
| 4 | 출고 처리 | CONFIRMED 주문 출고 실행 |
| 5 | 생산 라인 | 생산 현황 및 대기 큐 확인 |

---

## 기능 요구사항

### 1. 시료 관리

| 기능 | 입력 | 설명 |
|------|------|------|
| 시료 등록 | 시료 ID, 이름, 평균 생산시간, 수율 | 중복 ID 불가 |
| 시료 목록 조회 | - | 등록된 시료 전체 + 현재 재고 수량 표시 |
| 시료 이름 검색 | 검색어 | 이름 부분 일치 검색 |

> 수율 예시: 100개 생산 중 정상 90개 → 수율 0.9

---

### 2. 주문 관리

| 기능 | 입력 | 설명 |
|------|------|------|
| 주문 접수 | 시료 ID, 고객명, 주문 수량 | 생성 시 상태: `RESERVED` |
| 접수 주문 목록 | - | `RESERVED` 상태 주문 목록 표시 |
| 주문 승인 | 주문 ID | 재고 상황에 따라 자동 분기 (아래 참고) |
| 주문 거절 | 주문 ID | 즉시 `REJECTED` 전환 |

**승인 시 자동 분기**

```
재고 >= 주문 수량  →  재고 차감 후 CONFIRMED
재고 <  주문 수량  →  생산 라인 등록 후 PRODUCING
```

---

### 3. 모니터링

| 기능 | 표시 내용 |
|------|-----------|
| 주문 현황 | 상태별(`RESERVED` / `PRODUCING` / `CONFIRMED` / `RELEASE`) 주문 목록 (`REJECTED` 제외) |
| 재고 현황 | 시료별 재고 수량 + 재고 상태 |

**재고 상태 기준** (RESERVED 주문 수량 합계 대비)

| 상태 | 조건 |
|------|------|
| 고갈 | 재고 == 0 **(최우선 판단)** |
| 부족 | 0 < 재고 < RESERVED 주문 수량 합계 |
| 여유 | 재고 ≥ RESERVED 주문 수량 합계 |

> 고갈 조건이 여유/부족보다 우선 적용된다. 재고가 0이면 RESERVED 주문 유무와 무관하게 고갈로 표시한다.

---

### 4. 출고 처리

| 기능 | 입력 | 설명 |
|------|------|------|
| 출고 대기 목록 | - | `CONFIRMED` 상태 주문 목록 표시 |
| 출고 실행 | 주문 ID | `CONFIRMED` → `RELEASE` 전환 |

---

### 5. 생산 라인

| 기능 | 표시 내용 |
|------|-----------|
| 생산 현황 | 현재 생산 중인 Job 정보 (시료명, 실 생산량, 현재 생산량, 잔여 시간 등) |
| 대기 주문 목록 | 생산 큐에서 대기 중인 작업 목록 (FIFO 순서) |

**생산량 계산**

```
실 생산량  = ceil(부족분 / (수율 × 0.9))
총 생산시간 = 평균 생산시간 × 실 생산량
```

- 생산 완료 시 주문 상태: `PRODUCING` → `CONFIRMED`
- 스케줄링 전략: **FIFO** (선입선출)
