# Phase 5. 통합 테스트 및 검증

## 목표

5개의 시나리오로 전체 흐름을 검증하고, 엣지 케이스를 확인한다.

---

## 시나리오 1. 재고 충분 경로 (정상 흐름)

**목적**: 재고가 충분할 때 주문 → 승인 → 출고 흐름 검증

```
[준비]
  시료 등록: 이름="SiC-MOSFET", 평균생산시간=5, 수율=0.9, 초기재고=20

[실행]
  1. 주문 접수: 시료=SiC-MOSFET, 고객=A사, 수량=10
  2. 주문 승인
  3. 출고 실행

[검증]
  - 승인 후 상태: CONFIRMED
  - 재고: 20 - 10 = 10
  - 출고 후 상태: RELEASE
  - 모니터링 주문 현황: RELEASE 1건
  - 모니터링 재고: stock=10, 상태=여유 (RESERVED 없음)
```

---

## 시나리오 2. 재고 부족 경로 (생산 포함)

**목적**: 재고 부족 시 생산 → CONFIRMED 전환 흐름 검증

```
[준비]
  시료 등록: 이름="GaN-HPA", 평균생산시간=2, 수율=0.8, 초기재고=3

[실행]
  1. 주문 접수: 시료=GaN-HPA, 고객=B연구소, 수량=10
  2. 주문 승인

[검증 - 승인 직후]
  - 상태: PRODUCING
  - 부족분: 10 - 3 = 7
  - 실 생산량: ceil(7 / (0.8 * 0.9)) = ceil(9.72) = 10
  - 총 생산시간: 2 * 10 = 20 초
  - 모니터링 재고: stock=3, 상태=여유 (RESERVED 주문 없음 → RESERVED 합계=0, stock≥0)

  [20초 경과 후]
  - tick() 호출 시 생산 완료 감지
  - 재고: 3 + 10 - 10 = 3 (생산 추가 후 주문 차감)
  - 상태: CONFIRMED

[실행]
  3. 출고 실행

[검증 - 출고 후]
  - 상태: RELEASE
```

---

## 시나리오 3. 주문 거절 경로

**목적**: 거절된 주문이 모니터링에 노출되지 않는지 검증

```
[준비]
  시료 등록: 이름="InP-HBT", 평균생산시간=3, 수율=0.85, 초기재고=5

[실행]
  1. 주문 접수: 수량=2 → 주문 ID=1
  2. 주문 거절

[검증]
  - 상태: REJECTED
  - 모니터링 주문 현황: RESERVED/CONFIRMED/PRODUCING/RELEASE 모두 0건
  - REJECTED 주문은 어떤 목록에도 표시 안 됨
```

---

## 시나리오 4. 생산 큐 FIFO 순서 검증

**목적**: 다수의 PRODUCING 주문이 큐에 등록될 때 FIFO 순서로 처리되는지 검증

```
[준비]
  시료 등록: 이름="SiGe-BiCMOS", 평균생산시간=1, 수율=0.9, 초기재고=0

[실행]
  1. 주문1 접수: 수량=5 → 승인 → PRODUCING (shortage=5, targetQty=ceil(5/0.81)=7)
  2. 주문2 접수: 수량=3 → 승인 → PRODUCING (shortage=3, targetQty=ceil(3/0.81)=4)
  3. 주문3 접수: 수량=4 → 승인 → PRODUCING (shortage=4, targetQty=ceil(4/0.81)=5)

[검증 - 생산 대기 큐]
  - 현재 생산 중: 주문1 (7개, 7초)
  - 대기 큐: 주문2 → 주문3 순서

[검증 - 7초 경과 후]
  - 주문1: CONFIRMED
  - 현재 생산 중: 주문2

[검증 - 7 + 4초 경과 후]
  - 주문2: CONFIRMED
  - 현재 생산 중: 주문3
```

## 시나리오 5. 생산 진행률 중간 확인

**목적**: 생산 중간에 생산 현황 메뉴를 조회하면 진행률이 올바르게 표시되는지 검증

```
[준비]
  시료 등록: 이름="GaAs-pHEMT", 평균생산시간=5, 수율=0.9, 초기재고=0

[실행]
  1. 주문 접수: 수량=9 → 승인
     - shortage=9, targetQty=ceil(9/0.81)=12, totalTime=60초

[검증 - 10초 경과 후 생산 현황 조회]
  - producedQty = floor(10 / 5) = 2
  - 잔여 시간 = 60 - 10 = 50초

[검증 - 30초 경과 후]
  - producedQty = floor(30 / 5) = 6
  - 잔여 시간 = 30초

[검증 - 60초 경과 후]
  - producedQty = 12 (targetQty 도달)
  - tick() → CONFIRMED 전환
```

## 엣지 케이스

| 케이스 | 예상 동작 |
|--------|-----------|
| 존재하지 않는 시료 ID로 주문 접수 | "존재하지 않는 시료 ID" 메시지, 주문 생성 안 됨 |
| RESERVED 아닌 주문 승인 시도 | "승인 불가" 메시지, 상태 변경 안 됨 |
| CONFIRMED 아닌 주문 출고 시도 | "출고 불가" 메시지, 상태 변경 안 됨 |
| 수량 0 이하 입력 | 재입력 요청 |
| 수율 범위 초과 (0 이하 또는 1 초과) | 재입력 요청 |
| 생산 중 없는 상태에서 생산 현황 조회 | "현재 생산 중인 작업이 없습니다" 출력 |
| 이미 PRODUCING 인 시료에 추가 주문 승인 | 큐에 정상 추가, FIFO 대기 |
