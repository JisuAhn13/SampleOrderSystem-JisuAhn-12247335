# SampleOrderSystem — 반도체 시료 주문 관리 시스템

## 배경

가상의 반도체 회사 **S Semi**는 연구소, 팹리스(Fabless) 업체, 대학 연구실 등의 고객에게 반도체 시료를 납품합니다.
시료는 주문이 들어오면 웨이퍼 공정 설비를 통해 제작되고, 검수를 거쳐 고객에게 출고됩니다.
본 시스템은 **콘솔 기반 C++ 애플리케이션**으로 동작하며, 담당자가 직접 메뉴를 선택하여 시료를 등록하고 주문을 처리합니다.

| 역할 | 담당 업무 |
|------|-----------|
| **고객** | 이메일로 시료 요청 |
| **주문담당자** | 요청에 맞게 주문서 작성 및 등록 |
| **생산담당자** | 주문 수신 후 승인 또는 거절, 재고 확인, 출고 준비, 출고 처리 |
| **생산라인** | 재고 부족 시 시료 생산 |

---

## 주문 상태 흐름

```
RESERVED → (승인) → 재고 충분 → CONFIRMED → RELEASE
RESERVED → (승인) → 재고 부족 → PRODUCING → CONFIRMED → RELEASE
RESERVED → (거절) → REJECTED  (모니터링 제외)
```

| 상태 | 의미 |
|------|------|
| `RESERVED` | 주문 접수됨 |
| `REJECTED` | 주문 거절됨 |
| `PRODUCING` | 주문 승인 완료 및 재고 부족으로 생산 중 |
| `CONFIRMED` | 주문 승인 완료 및 출고 대기 중 |
| `RELEASE` | 출고 완료 |

## 메인 메뉴

| 번호 | 메뉴 | 설명 |
|------|------|------|
| 1 | **시료 관리** | 새로운 시료 등록, 목록 조회, 이름 검색 |

| 3 | **모니터링** | 상태별 주문 수 및 시료별 재고 현황 확인 |
| 4 | **출고 처리** | `CONFIRMED` 상태 주문에 대해 출고 실행 |
| 5 | **생산 라인** | 현재 생산 중인 시료 및 대기 중인 생산 큐 확인 |

## 1. 시료 관리

- **시료 등록**: 새로운 시료를 시스템에 추가
  - 속성: 시료 ID (자동 채번), 이름, 평균 생산시간, 수율, 초기 재고
- **시료 조회**: 등록된 모든 시료 목록 + 현재 재고 수량 확인
- **시료 검색**: 이름으로 특정 시료 검색

> **수율** = 정상 시료 수 / 총 생산 시료 수 (0.0 ~ 1.0)
> 예) 100개 생산 중 정상 90개 → 수율 0.9

---

## 2. 주문 관리

고객이 시료를 요청하면 주문담당자가 주문을 생성합니다.

### 시료 예약
- 고객이 원하는 시료와 수량을 주문
- 입력값: 시료 ID, 고객명, 주문 수량
- 주문 생성 시 초기 상태: `RESERVED`

### 접수된 주문 목록
- `RESERVED` 상태의 주문 목록 표시

### 주문 승인
- 특정 주문에 대해 승인 처리 → 재고 상황에 따라 자동 분기
  - 재고 **충분** → 즉시 `CONFIRMED` 상태로 전환, 재고 차감
  - 재고 **부족** → 생산 라인에 자동 등록 + `PRODUCING` 상태로 전환

### 주문 거절
- 특정 주문에 대해 거절 처리 → 즉시 `REJECTED` 상태로 전환

---

## 3. 모니터링

담당자가 현재 시스템 상태를 한눈에 파악할 수 있도록 구성합니다.

- **주문량 확인**: 상태별(`RESERVED` / `CONFIRMED` / `PRODUCING` / `RELEASE`) 주문 목록 표시 (`REJECTED` 제외)
- **재고량 확인**: 시료별 현재 재고 수량 및 상태 표기

| 재고 상태 | 의미 |
|-----------|------|
| 여유 | 재고 ≥ 접수된(RESERVED) 주문 수량 합계 |
| 부족 | 0 < 재고 < 접수된(RESERVED) 주문 수량 합계 |
| 고갈 | 재고 수량이 0인 상태 |

---

## 4. 생산 라인

재고 부족분을 생산하되, 수율 및 오차를 고려하여 생산량을 결정합니다.

### 생산량 계산

| 항목 | 계산식 |
|------|--------|
| **실 생산량** | `ceil(부족분 / (수율 × 0.9))` |
| **총 생산 시간** | `평균 생산시간 × 실 생산량` |

### 생산 시뮬레이션 방식

- **시간 기반 순차 처리**: 생산 큐(FIFO)의 첫 번째 Job만 진행, 완료 후 다음 Job 자동 시작
- 조회 시점에 `경과 시간 / 평균 생산시간`으로 현재 생산량(`producedQty`) 동적 계산
- `producedQty >= 실 생산량` 이면 자동으로 `PRODUCING → CONFIRMED` 전환 및 재고 반영
- 스케줄링 전략: **FIFO** (선입선출)

```
현재 생산량 = floor(경과 시간 / 평균 생산시간)
```

### 표시 정보
- **생산 현황**: 현재 생산 중인 시료 정보 (시료명, 실 생산량, 현재 생산량, 잔여 시간 등)
- **대기 주문**: 생산 큐에서 대기 중인 작업 목록 (FIFO 순서)

---

## 5. 출고 처리

- `CONFIRMED` 상태의 주문 목록 표시
- 출고 실행 → 주문 상태: `CONFIRMED` → `RELEASE`

## 클래스 구조

```
Sample          — 시료 정보 (ID, 이름, 평균 생산시간, 수율, 재고)
Order           — 주문 정보 (주문 ID, 시료, 고객명, 수량, 상태)
ProductionJob   — 생산 작업 (대상 Order, 목표 생산량, 현재 생산량, 생산 시작 시각)
SampleManager   — 시료 CRUD, 재고 관리
OrderManager    — 주문 접수, 승인, 거절
ProductionLine  — 생산 큐(FIFO) 관리, 시간 기반 생산 진행
Monitor         — 주문 현황, 재고 현황 조회
ShippingManager — 출고 처리
MenuUI          — 콘솔 메뉴 출력 및 입력 처리
```
